1. users — Пользователи Telegram-бота и android приложения.
Хранит информацию о каждом пользователе сервиса.
id — Уникальный внутренний идентификатор пользователя.
telegram_id — ID пользователя в Telegram (уникален, опционален).
username, first_name, last_name — Данные из Telegram (если доступны).
gender — Пол пользователя (опционально).
created_at — Дата регистрации.
2. chat_sessions — Сессии общения с ИИ
Каждая сессия — это один диалог, где сохраняется история сообщений.
id — Уникальный ID сессии.
user_id — Ссылка на пользователя (users.id).
started_at — Время начала сессии.
ended_at — Время окончания (если сессия завершена).
is_active — Открыта ли сессия сейчас.
3. messages — Сообщения в сессии
Содержит все входящие и исходящие текстовые сообщения.
id — Уникальный ID сообщения.
session_id — Ссылка на сессию (chat_sessions.id).
sender — Кто отправил сообщение: user или ai.
message_text — Текст сообщения.
created_at — Время отправки.
token_usage — Количество токенов, использованных при генерации ответа (для AI).
4. ai_requests — Журнал AI-запросов
Хранит техническую информацию о запросах к языковой модели.
id — Уникальный ID запроса.
message_id — Ссылка на сообщение (messages.id).
request_payload — JSON-тело запроса к AI.
response_payload — JSON-тело ответа от AI.
response_time_ms — Время ответа (в миллисекундах).
status_code — HTTP-статус или внутренний статус.
error_message — Ошибка, если возникла.
created_at — Дата и время запроса.
5. subscriptions — Подписки пользователей
Хранит информацию о тарифах и лимитах доступа.
id — Уникальный ID подписки.
user_id — Ссылка на пользователя (users.id).
plan_name — Название плана (например, "free", "premium").
is_active — Активна ли подписка.
start_date, end_date — Срок действия.
usage_limit — Лимит запросов или токенов.
used_requests — Уже использованные запросы.
auto_renew — Будет ли подписка автоматически продлеваться.
6. usage_logs — Лог использования сервиса
Агрегированная статистика по пользователю за день.
id — Уникальный ID записи.
user_id — Ссылка на пользователя (users.id).
date — Дата.
requests_count — Сколько запросов сделал пользователь.
tokens_used — Сколько токенов было потрачено.
session_count — Количество сессий за день.
7. admins — Администраторы сервиса
Управление ролями и доступами внутри админки.
id — Уникальный ID записи.
user_id — Ссылка на пользователя (users.id).
role — Роль администратора (owner, admin, moderator).
created_at — Когда был назначен админом.



С front платформы дёргается метод, в параметры которого суётся модель, содержащая поля: имя, пол, возраст
